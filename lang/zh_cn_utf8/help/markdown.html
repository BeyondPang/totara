使用标记符增强文本

通过允许您轻松添加强调（粗体、斜体），结构（点句和标题）和（图像或其他 web 资源的）链接，标记符增加了 Moodle 中的简单文本输入格式。在 Moodle 中的很多地方都可以使用标记符，只需要从格式设置下拉列表中选择标记符，该列表位于可在其中进行选择的文本输入区域。

基本概念

要输入文本，只需要将其输入在文本输入区域或文本框中，并在一个段落末尾按两下“返回”键，在一段末尾和下一段开头留出一个空白行。

强调文本

您可以使用标记符添加三个级别的强调，即斜体 文本、粗体文本或粗斜体 文本可以通过在您想要强调的文本前后加上星号来实现，例如：

*斜体* -> 斜体
**粗体** -> 粗体
***粗斜体*** -> 粗斜体

强调可以添加给单个词，连续几个词或是词的某部分：

*单个* 词 -> 单个词
***连续几个词*** -> 连续几个词
不能**区别**的 -> 不能区别的

可以交替使用下划线 (_) 和星号来表示强调。

标题

标记符允许您使用标题细分文本，它有六个不同的细分级别，但是普通文本很少使用三个以上级别。例如，在当前阅读的文本中使用了三个级别的标题。

您可以创建一个标题，以一个或多个哈希字符 (#)开始一行。一个哈希字符是最大且最重要的标题，而六个则表示标题最不重要或最小。

# 一级章节标题
## 二级章节标题
### 三级章节标题
等

前两个级别的标题是最常见的，可以用两种方法来创建，使其在文本版本中突显出来（但是输出与前一种方法完全相同）。这两种法是在标题下划一行等号 (=) 或连字符 (-)。例如：

章节标题
===============

子章节标题
------------------

列表

点句列表

可以通过以星号开始每一行，并且在点句内容前加一个空格的方式来创建点句列表。请注意，空格很重要，一定不能遗漏。

* 第一点
* 第二点
* 第三点

变为

  • 第一点
  • 第二点
  • 第三点

编号列表

同样地，可以通过以一个数字开始每一行，并且在相关文本前加一个空格的方式来创建编号列表。

1. 第一点
2. 第二点
3. 第三点

变为

  1. 第一点
  2. 第二点
  3. 第三点

缩进式列表

您可以将点句列表和编号列表按大小顺序套叠在一起，或者将他们缩排，还可以将点句和编号列表组合在同一个结构中：

* 最高级点句一
* 子点句
* 子点句 2
* 最高级点句二
1. 编号点一
1. 套叠的编号点
2. 编号点二

变为

  • 最高级点句
    • 子点句
    • 子点句 2
  • 最高级点句二
    1. 编号点一
      1. 套叠的编号点
    2. 编号点二

引用段落

您可以通过以一个尖括号 (>) 开始每一行来表示文本的引用部分。很多电子邮件程序都选择用此符号来表示引用部分。但输出时引用部分就变成两端缩进。

> 这是一个引用段落
> 分为两行

变为

这是一个分为两行的引用段落

您可以省去一些输入，只在该段第一行的开始处使用一个尖括号

> 这全部是一个单独引用的
段落，即使它分为
几行。

变为

这全部是一个单独引用的段落，即使它分为几行。

Web 链接

有两种方法可以创建 web 资源链接：第一种方法是包含嵌入式链接，将您想要读者单击的文本放在方括号中，紧接着将此页的 URL 放在其后的圆括号中，圆括号和前面的方括号之间不留空格或间隔。您还可以为此链接添加可选标题，放在 URL 后引号内。

[示例链接]（http://example.com/“可选标题”）在一个句子中。

变为

示例链接 在一个句子中。

如果提供标题，则标题会显示在“工具提示”中，用户将鼠标光标置于链接文本上时就会出现“工具提示”。试用上面出现的链接。

对于较长的链接，您可以使用脚注形式来避免影响文本连贯性，即在第二对方括号中附加一个此链接的简短定义名称（或者使用简短的说明性词、短语或仅仅是一个数字）。

[示例链接][ex]在一个句子中。

这样，在文档的任何位置，更好的处理方法是直接使链接紧跟段落之后，或者将所有链接 URL 集合起来放在文档末尾，您可以使用 id 定义关联的 URL：

[ex]：http://example.com/“可选标题”

第二个示例的输出与第一个示例一样。以上方法使您可以轻松保持工作文档的简洁，有助于进一步编辑（尤其是当您和其他人一起工作的时候）。

最后一个快捷方法：如果您想要您的链接文本和 URL 一样，则将 URL 放在尖括号里，如下：

<http://moodle.org/>

变成

http://moodle.org

图像

图像的处理方式与 web 链接很相似，不同的是要在图像前面加一个感叹号。“alt 文本”（alt 意思是替代）是提供给因各种原因无法查看图像的用户的解释文本，因此此文本在没有任何视觉提示的情况下应该清楚解释该图像。这么做也就此图像的目的，给那些用标记符编辑文本的人提供一种提醒或提示。

当用户将光标置于图像上时，“标题”将显示在一个小的弹出窗口中，因此可以提供一些附加信息。

![alt 文本]（/path/img.jpg“可选标题”）

与 web 链接一样，可以通过使用“脚注”形式的引用将所有 URL 放在一起的方式来组织您的文档。只给图像一个简短而唯一的名称：

![alt 文本][照片]

在文档的任何位置，将该名称与图像文件相关联：

[照片]：/url/to/img.jpg“可选标题”

以下是一个示例，后面是其输出：

![Google 徽标]（http://www.google.com/images/logo.gif“Google 徽标"）

“alt 文本”因其辅助功能也很重要，图像链接损坏或是暂时无法查看图像的浏览者将会经常使用此文本：

![断开的图像的 alt 文本]（http://example.com/intentionally.broken.link“不再显示此图像”）

变成（确切的输出取决于您的浏览器）
